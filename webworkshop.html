3 Om WordPress

3.1 Förberedelser:

  • Ladda ner Bitnami WordPress Stack och installera (om du stöter på problem så löser vi det på workshopen). Det är detta program vi använder för att WordPress ska köra lokalt. Det fungerar så att bitnami sköter en databas (där all data, t.ex. text och inställningar sparas) och en server (en Apache-server som hämtar och visar innehåll för oss på rätt sätt). Det fungerar på samma sätt när en WordPress-sida slutligen sätts i drift.
  • När bitnami installerats så kommer den att öppna ett fönster i din webläsare. Välj att gå vidare till din WordPress-stack (det ska finnas en stor blå länk högst upp på sidan som du kan trycka på!) I adressfönstret på webläsaren (högst upp i mitten vanligast) står nu en IP-adress, t.ex. "http://127.0.0.1:8080/wordpress/". Det är din bas-URL (i brist på bättre svenska ord :) ) och jag kommer att skriva den som [bas-url] från och med nu, så får du sätta in "XXX.X.X.X:XXXX/wordpress" eller vad det nu står helt själv! Gå in på "[bas-url]/wp-admin/" och du är i admin-gränssnittet
  • Hitta var bitnami installerats och gå in i undermappen "apps/wordpress/htdocs". Det är denna mapp som innehåller alla WordPress filer. Bokmärk eller skriv ner hur du hittar tillbaka till den här mappen!

3.2 Hur WordPress fungerar (kortfattat)

Det här är viktigt att förstå lite av för att förstå sammanhanget i vilket vi gör webbsidor. WordPress har ett administrationsgränssnitt. I det skapar användaren innehåll som sparas i en MySQL-databas. När en besökare navigerar till sidan kör servern koden i WordPress. Koden producerar HTML, CSS och javascript, vilken användarens webbläsare tar emot, bearbetar och visar. WordPress PHP-filer innehåller en blandning av HTML och PHP-kod som körs och (i slutändan) producerar HTML-kod. PHP-koden ligger innanför taggarna <?php … ?>. Den viktigaste mappen är undermappen "wp-content". Däri ligger (bland annat) följande undermappar:

  • "themes" - innehåller teman dvs. olika utseenden för sidan eller bloggen. Mappen innehåller olika teman i separata mappar, t.ex. "twentyeleven" som är standardtemat för WordPress
  • "plugins" - innehåller plugins dvs. program som lägger till extra funktioner till temat, till exempel "akismet" som skyddar WordPress kommentarsfält mot spam
  • "languages" - innehåller filer för att visa text på olika språk

3.3 Temats filer

Ett tema innehåller minimalt en enda fil: style.css. Filen innehåller CSS och dessutom information om temat som visas i WordPress administrationsgränssnitt. Förutom denna fil innehåller temat filen functions.php vilken innehåller PHP-funktioner som kallas i andra tema-filer eller för att förändra beteendet hos hela temat, t.ex. lägga till funktionalitet som valbara headerbilder. För att visa olika vyer innehåller teman också olika templatefiler, exempelvis:

  • index.php - innehåller kod för att visa 'startsidan', normalt en lista av de senaste inläggen
  • header.php - inleder HTML-dokumentet med <html>, <head>…</head>, <body> och ett sidhuvud med rubrik och navigationsmeny
  • footer.php - avslutar HTML-dokumentet med en sidfot, </body> och </html>
  • sidebar.php - visar sidebar-menyn
  • single.php - för att visa enskilda inlägg
  • page.php - för att visa 'sidor' (speciella WordPress-inlägg som är till för att presentera mer statisk information än blogginlägg)
  • comments.php - för att visa kommentarsfält

Kolla igenom de här filerna i temat twentyeleven och försök begripa vad du kan. Skriv ner eller hjälp varandra med de PHP-bitar som är svårförståeliga.

3.4 Admingränssnittet

Administrationsgränssnittet, som finns på "[bas-url]/wp-admin/", är bara tillgängligt för inloggade användare. Där skriver du nya inlägg, byter tema, hanterar tillägg, och annat. Gränssnittet borde vara lätt att begripa för alla som bloggat tidigare, så vi fördjupar oss inte i det här.

En funktion kan ha argument, det vill säga information som måste skickas till den när den kallas.   • en viktig typ av struktur i PHP är en array som är en datastruktur som innehåller ett antal värden med varsin "nyckel" som måste vara ett heltal eller en sträng. Arrays definieras genom funktionen array. Exempel:
  • vanligt förekommande är också:
    • if (<condition>) {<consequence>} - om <condition> gäller så körs koden inom måsvingarna. Det kan också skrivas if (<condition>) : <consequence> endif;
    • varianter på koden ovan: if (<condition>) {<consequence>} else {<something else>} eller if (<condition>) {<consequence>} elif (<other condition>) {<something else>} else {…}
    • while (<condition>) {<consequence>} - så länge som <condition gäller körs koden i måsvingarna om och om igen, och varje gång testar while-loopen om <condition> fortfarande är uppfyllt. Kan även skrivas while (<condition>) : <consequence> endwhile;

4 Att bygga ett tema

Det absolut enklaste sättet att bygga en hemsida med WordPress är genom att modifiera ett redan existerande tema. Det mest effektiva sättet att göra det är genom att använda WordPress Child Themes. Du kan använda dig av ett bra tema eller en tema-mall, och gradvis ändra och bygga på funktioner beroende på vad som passar dig, vilket sparar tid och håller dig borta från kod du inte vill se eller modifiera för tillfället. Det har också fördelen att du kan fortsätta använda dig av uppdaterade versioner av samma tema, men behålla dina ändringar utan att manuellt behöva göra om ändringarna i den nya versionen.

4.2.1 Allmän layout

CSS kan vara oförutsägbart. En bra idé är att dela in sidan i ett rutnät av block av olika storlek, en grid. På så sätt kan innehåll placeras och hamnar automatiskt i tydliga avdelningar. Det är att rekommendera att använda taggarna:

  • <header>…</header> - för sidhuvud
  • <section>…</section> - för olika funktionellt olika sektioner, t.ex. bildspel, senaste inlägg, sociala medier osv.
  • <div>…</div> - ett alternativ till <section>, framför allt för indelningar som krävs för att få layouten rätt
  • <aside>…</aside> - för sidebar och sidoställt innehåll i sektioner
  • <nav>…</nav> - för navigationsmeny
  • <article>…</article> - för att visa inlägg och utdrag ur inlägg

4.4.2 Funktionen add_theme_support($feature)

Genom att kalla denna funktion i functions.php kan vi lägga till funktionalitet till temat. Funktionen har ett antal olika argument:

  • 'post-formats' - inlägg kan väljas i flera format - bildgallerier, videos, statusuppdateringar, podcasts
  • 'post-thumbnails' - en utvald bild kan läggas till varje inlägg
  • 'custom-background' - valbar bakgrundsfärg och bakgrundsbild
  • 'custom-header' - möjlighet att välja en bild som normalt visas i sidhuvudet genom att kalla funktionen header_image()
  • 'automatic-feed-links' - lägger till feedlänkar till alla relevanta RSS/Atom-feeds för den aktuella sidan som visas
  • 'menus' - aktiverar valbara navigationsmenyer

En del av de här extrafunktionerna kan inte användas rakt av, utan kräver lite extrajobb. Se dokumentationen till funktionen!

5 Domän och server

En kort sammanfattning av vad som krävs för att sätta upp en WordPressinstallation.

5.1 Att köpa en domän

Du kommer behöva köpa en domän, det kan du göra på http://loopia.se. Kostnaden bör inte vara större än någon hundralapp.

5.2 Att välja och köpa en serverplats

Kolla runt på många olika alternativ när du letar efter en server. Kanske har du redan tillgång till en, eller kan sätta upp en själv på en stationär dator du har stående hemma. Det är bekvämt speciellt för nybörjaren att skaffa ett webhotell till skillnad från en vanlig server. Den senare är ett användarkonto på en dator, till vilken du ansluter med FTP eller ssh och blir tvungen att sköta Apache och MySQL-databaser själv. Det tidigare erbjuder ett gränssnitt där du kan lägga till databaser, navigera grafiskt i en mappstruktur, sköta mail med mera. Ett minimalt webhotell erbjuds av arbetarkollektivet bakom http://trick.ca - du kommer undan billigt men överväg faktorerna:

  • bandbredd - hur många MB trafik/månad tillåter servern?
  • lagringsutrymme - en WordPressinstallation ligger runt 20 MB, men mediafiler tar mycket plats. Om ni behöver plats för många bilder eller om ni lagrar egna videofiler kan det krävas mycket utrymme.

5.3 Att installera WordPress

För det första, kolla upp referensmaterialet till hur WordPress installeras på http://codex.wordpress.org/Installing_WordPress. Du behöver göra följande:

  • ladda upp innehållet i mappen htdocs, d.v.s. alla WordPress filer, på din server
  • skapa en MySQL-databas och en användare
  • göra ändringar i filen wp-config.php (använd dig av mallen wp-config-sample.php) för att ange uppgifter till databasen du just skapade
  • om du lagt WordPress filer på rätt ställe så kan du köra installationsscriptet genom att gå till adressen wp-admin/install.php. Resten sköts automagiskt.

Många servrar har ett gränssnitt som heter CPanel. Det innehåller ett verktyg för databaser som heter phpMyAdmin som du kan använda för att skapa databasen enkelt. Leta efter information själv, ingen av stegen ovan ska vara vidare svåra att lära sig själv!
